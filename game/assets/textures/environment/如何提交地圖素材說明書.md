# 地圖素材提交說明書（美術組專用）

本文件說明如何製作與提交場景地圖相關的素材（Tileset / 裝飾物），讓程式組能直接在 Godot 中使用。

---

## 📦 兩種素材類型

場景地圖由兩類素材組成，**分別存放在不同資料夾**：

| 類型 | 用途 | 範例 | 存放位置 |
| --- | --- | --- | --- |
| **Tileset 拼貼圖** | 重複使用的地面、牆、柵欄 | 磚牆、草地、石板路 | `tilesets/<區域>/` |
| **Prop 裝飾物** | 獨立擺放的大型物件 | 樹、長椅、路燈、公告欄 | `props/<類別>/` |

**判斷原則：**
- 會在地圖上**重複拼貼**（超過 4 次以上）→ Tileset
- 單獨擺放、**尺寸不規則**、可能**會擋住角色** → Prop

---

## 📂 資料夾結構

```text
game/assets/textures/environment/
├── tilesets/
│   ├── nccu/         # 政大校園
│   ├── market/       # 木柵市場
│   ├── riverside/    # 道南河濱
│   └── zhinan/       # 指南山
│
└── props/
    ├── nature/       # 自然類：樹、灌木、花圃、草叢
    └── urban/        # 人造類：長椅、路燈、公告欄、垃圾桶、欄杆
```

請依照**場景**（tilesets）或**類別**（props）將檔案放入對應資料夾。

---

## 🎨 Tileset 製作規格

### 尺寸要求

- **每格 tile 尺寸：32×32 像素**（與角色相同像素風格）
- 整張 tileset PNG **寬度必須是 32 的倍數**、高度必須是 32 的倍數
- 建議單張 tileset 不超過 2048×2048，避免記憶體壓力

> ⚠️ 若使用其他 tile 尺寸（例：16×16 或 64×64），請先與程式組確認。

### 排版要求

- 所有 tile **對齊 32 像素網格**（每格左上角座標必須是 32 的倍數）
- 相鄰 tile 之間**不留間距**（no padding）
- 透明區域請保留為完全透明（alpha = 0），不要填白或填黑

### 檔名規範

```text
<場景>_<主題>.png

範例：
nccu_campus_v1.png          # 政大校園主 tileset
nccu_walls.png              # 政大牆體與柵欄
market_stalls.png           # 市場攤位
```

## 🌳 Prop 製作規格

### 尺寸要求

- **無固定尺寸**，但建議高度不超過 192 像素（6 個 tile 高），避免遮擋過多畫面
- 單張 PNG **只放一個 Prop**（不要一張圖塞多個物件）
- 背景**完全透明**

### 錨點對齊（重要！）

Prop 會使用 Y 軸排序，判定遮擋用的「腳底」必須對齊到圖片底部中央：

```text
┌──────────────┐
│              │
│    🌳         │   ← Prop 圖像
│              │
│              │
│──────⚓──────│   ← 腳底錨點：圖片底部中央
└──────────────┘
```

若 Prop 底部有陰影或碎屑（如樹葉落地），請將陰影**置中**。

### 檔名規範

```text
<物件名稱>_<編號>.png

範例：
tree_oak_01.png
tree_pine_02.png
bench_wood.png
lamp_street_01.png
bulletin_board_cork.png
```

#### 🚫 檔名硬性規則（違反會被自動拒絕）

**資產檔名必須純 ASCII**。**不可**使用：

| 類型 | 範例 | 原因 |
| --- | --- | --- |
| 中文 | `樹.png`、`長椅_01.png` | GitHub 跨平台 checkout 亂碼 |
| 日文/韓文 | `みどり.png` | 同上 |
| Emoji | `tree🌳.png` | Godot import 會失敗 |
| 全形符號 | `tree　01.png`（全形空白） | Python 讀檔不可靠 |
| 半形空白 | `my file.png` | Shell 與 URL 處理麻煩 |
| 特殊符號 | `tree@home.png`、`a/b.png` | 檔系統非法字元 |

✅ **只允許：**`a-z` `A-Z` `0-9` `_`（底線） `-`（連字號） `.`（點）

✅ **必須以字母或數字開頭**（不可以 `.` 或 `-` 起頭）

GUI 拆圖工具會在匯出前自動驗證，違規會跳出錯誤無法通過。

## 💡 通用注意事項

### 像素風格

- **Filter Mode 必須是 Nearest**（在 Godot 中已設為全局預設，不需手動設定）
- 不要使用抗鋸齒、模糊、漸層柔邊（會讓像素風格破功）
- 色板建議維持在 32 色以內（與角色風格一致）

### 透明度

- 所有素材**必須有 alpha 通道**（RGBA PNG）
- 邊緣不要殘留淡灰色（常見於 Photoshop 匯出時的殘影）

### 視角

- **Low Top-down（微俯視）**，與角色視角一致
- Tileset 的地面為**平面俯視**，牆體與 Prop 為**微側視**

## 🤖 自動拆圖工具（推薦）

如果你習慣在一張大畫布上畫所有素材（像這次的 university tileset 那樣混合了 tile + prop），**不用手動切圖**。我們提供了兩個版本：

### ⭐ GUI 視窗版（推薦給美術組）

視覺化介面，不需要記指令。執行：

```bash
python ai_engine/scripts/split_map_assets_gui.py
```

#### 📥 事前準備：用 temp/ 暫存區

把從繪圖軟體匯出的大圖集中丟到：

```
game/assets/textures/temp/
```

此資料夾已加入 `.gitignore`，你丟的原始大圖**不會誤 push 到 GitHub**。GUI 打開檔案選擇時會自動從這裡開始找，不用每次翻資料夾。

#### 🔍 透明度檢查（拆圖結果異常時用）

如果偵測後整張圖被判成**一個巨大 blob**（沒有分離出多個 prop），**99% 是透明度問題**：

- 加入檔案後，先選幾個檔按「🔍 檢查透明度」
- 診斷視窗顯示每張圖的 alpha 分布：
  - 🟢 **完全透明 (α=0)**：應 ≥ 30%（prop 之間的空隙）
  - 🟠 **半透明（軟邊）**：應 < 5%（太高代表邊緣羽化/陰影）
  - 🟦 完全不透明 (α=255)：prop 本體
- 有問題會列出具體原因與修正建議
- 可按「💾 匯出 alpha 預覽圖」存一張灰階圖：
  - 白色 = 實體像素
  - 黑色 = 透明背景
  - 灰色 = 軟邊/羽化 — **若看到灰色「橋」把不同 prop 連起來，就是拆圖失敗的原因**

按「偵測並進入標記」時，工具也會**自動預檢**：若透明度明顯有問題會先彈窗警告（避免白跑 15 秒 BFS）。

#### 🔄 批次模式（一次處理多張大圖）

如果你一次匯出了多張大圖（例如：tree_sheet.png、bench_sheet.png、lamp_sheet.png），不用一張一張跑工具——直接把它們一口氣加到清單裡批次處理：

**操作流程：**

1. **Setup 畫面**：
   - 點「➕ 加入檔案...」→ 選擇多個 PNG（按住 `Ctrl` 或 `Shift` 多選）
   - 選錯可按「➖ 移除所選」或「🗑 清空清單」
   - 選擇目標資料夾（通常是 `.../props/nature` 或 `.../props/urban`）
   - 按「偵測並進入標記 →」

2. **標記畫面**（每張大圖各自標記一輪）：
   - 頂部顯示「第 1/3 張：tree_sheet.png」指示進度
   - 每個 prop 顯示縮圖、編號、尺寸、名稱輸入框
   - ☐ 勾選要保留的，不要的取消勾選
   - 快捷鍵：
     - `Ctrl+A`：全選
     - `Ctrl+D`：全不選
     - `Ctrl+I`：反選
   - **批次前綴**：想一次命名一整組類似物件？輸入前綴 → 套用到全部（例：輸入 `tree`，勾選的 prop 就自動變成 `tree_001`、`tree_002`…）
   - 按「驗證並匯出 → 下一張」（最後一張時顯示「驗證並匯出（最後一張）」）
   - 不想處理這張？點「跳過此張」直接進下一張

3. **自動驗證**（若有問題才跳出對話框）：
   - ✗ 非法檔名（中文、日文、emoji、空白、斜線等） → 必須回去修正
   - ⚠ 目標資料夾已有同名檔 → 選擇：跳過 / 覆蓋 / 兩者都保留（自動加 `_2`）
   - ⚠ 像素完全相同的重複圖 → 選擇：跳過 / 覆蓋 / 兩者都保留（自動加 `_dup`）/ 刪除舊檔

4. **批次完成**：跑完最後一張後顯示總計匯出統計。

### CLI 指令版（給會用命令列的人）

### 使用方式

```bash
# 從專案根目錄執行
python ai_engine/scripts/split_map_assets.py \
    --input "你的大圖路徑/mega_sheet.png" \
    --tileset-region 0,0,2752,300 \
    --tileset-zone nccu \
    --tileset-name campus \
    --props-region 0,300,2752,1536 \
    --props-category urban
```

### 參數說明

| 參數 | 說明 |
| --- | --- |
| `--input` | 大圖 PNG 路徑 |
| `--tileset-region x,y,w,h` | 大圖中 tileset 拼貼區的矩形座標（左上 x, y + 寬高） |
| `--tileset-zone` | 此 tileset 屬於哪個場景：`nccu` / `market` / `riverside` / `zhinan` |
| `--tileset-name` | Tileset 命名後綴（例如 `campus` → 輸出為 `nccu_campus.png`） |
| `--props-region x,y,w,h` | 散落 prop 的區域（腳本會自動偵測透明背景中的物件） |
| `--props-category` | Prop 分類：`nature`（樹、花圃） / `urban`（路燈、長椅、公告欄） |
| `--dry-run` | 只預覽會產生什麼，不實際寫檔 |

### 腳本會自動做的事

1. ✂️ 把 tileset 區域裁切出來，存到 `tilesets/<zone>/<zone>_<name>.png`
2. 🔍 掃描 prop 區域內所有**非透明的連通區塊**，自動 bbox 裁切每個 prop
3. 📝 按閱讀順序（上到下、左到右）自動編號為 `prop_001.png`、`prop_002.png`…
4. 🖼️ 產生一張 `_preview.png` 標號總覽，讓你知道哪個編號是哪個物件

### 拆完之後

打開 `_preview.png`，對照編號把 `prop_003.png` 手動改名為 `bench_wood_01.png` 之類的有意義名稱（參考 [Prop 檔名規範](#檔名規範-1)）。

> ⚠️ 前提：prop 之間**必須有透明間距**，否則會被判定成同一個 blob。Tileset 拼貼區相反，**不能有間距**。所以建議大圖上：tileset 區集中放上方緊密排列，prop 散落在下方留空隙。

### 簡化版（只拆 prop）

如果大圖上只有散落的 prop、沒有 tileset：

```bash
python ai_engine/scripts/split_map_assets.py \
    --input props_only.png \
    --props-category urban
```


若不熟悉 Git，請：

2. 壓縮成 zip 傳給程式組
3. 附上簡短說明（新增了什麼、替換了什麼）

## 🔧 程式組端處理流程（僅供參考）

美術提交素材後，程式組會：

1. 將 tileset PNG 匯入 Godot，建立 `TileSet` 資源（`src/maps/tilesets/<zone>_tileset.tres`）
2. 為 Prop 建立對應的場景（`src/maps/props/<category>/<prop>.tscn`）
3. 在 Zone 場景中擺放 tile 與 prop

美術組**不需要**操作 Godot，只需提供素材檔案即可。
