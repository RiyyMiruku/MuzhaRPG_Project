# 地圖素材提交說明書（美術組專用）

本文件說明如何製作與提交場景地圖相關的素材（Tileset / 裝飾物），讓程式組能直接在 Godot 中使用。

---

## 📦 兩種素材類型

場景地圖由兩類素材組成，**分別存放在不同資料夾**：

| 類型 | 用途 | 範例 | 存放位置 |
| --- | --- | --- | --- |
| **Tileset 拼貼圖** | 重複使用的地面、牆、柵欄 | 磚牆、草地、石板路 | `tilesets/<區域>/` |
| **Prop 裝飾物** | 獨立擺放的大型物件 | 樹、長椅、路燈、公告欄 | `props/<類別>/` |

**判斷原則：**
- 會在地圖上**重複拼貼**（超過 4 次以上）→ Tileset
- 單獨擺放、**尺寸不規則**、可能**會擋住角色** → Prop

---

## 📂 資料夾結構

```text
game/assets/textures/environment/
├── tilesets/
│   ├── nccu/         # 政大校園
│   ├── market/       # 木柵市場
│   ├── riverside/    # 道南河濱
│   └── zhinan/       # 指南山
│
└── props/
    ├── nature/       # 自然類：樹、灌木、花圃、草叢
    └── urban/        # 人造類：長椅、路燈、公告欄、垃圾桶、欄杆
```

請依照**場景**（tilesets）或**類別**（props）將檔案放入對應資料夾。

---

## 🎨 Tileset 製作規格

### 尺寸要求

- **每格 tile 尺寸：32×32 像素**（與角色相同像素風格）
- 整張 tileset PNG **寬度必須是 32 的倍數**、高度必須是 32 的倍數
- 建議單張 tileset 不超過 2048×2048，避免記憶體壓力

> ⚠️ 若使用其他 tile 尺寸（例：16×16 或 64×64），請先與程式組確認。

### 排版要求

- 所有 tile **對齊 32 像素網格**（每格左上角座標必須是 32 的倍數）
- 相鄰 tile 之間**不留間距**（no padding）
- 透明區域請保留為完全透明（alpha = 0），不要填白或填黑

### 檔名規範

```text
<場景>_<主題>.png

範例：
nccu_campus_v1.png          # 政大校園主 tileset
nccu_walls.png              # 政大牆體與柵欄
market_stalls.png           # 市場攤位
```

---

## 🌳 Prop 製作規格

### 尺寸要求

- **無固定尺寸**，但建議高度不超過 192 像素（6 個 tile 高），避免遮擋過多畫面
- 單張 PNG **只放一個 Prop**（不要一張圖塞多個物件）
- 背景**完全透明**

### 錨點對齊（重要！）

Prop 會使用 Y 軸排序，判定遮擋用的「腳底」必須對齊到圖片底部中央：

```text
┌──────────────┐
│              │
│    🌳         │   ← Prop 圖像
│              │
│              │
│──────⚓──────│   ← 腳底錨點：圖片底部中央
└──────────────┘
```

若 Prop 底部有陰影或碎屑（如樹葉落地），請將陰影**置中**。

### 檔名規範

```text
<物件名稱>_<編號>.png

範例：
tree_oak_01.png
tree_pine_02.png
bench_wood.png
lamp_street_01.png
bulletin_board_cork.png
```

**不要用中文檔名**（避免跨平台編碼問題）。

## 💡 通用注意事項

### 像素風格

- **Filter Mode 必須是 Nearest**（在 Godot 中已設為全局預設，不需手動設定）
- 不要使用抗鋸齒、模糊、漸層柔邊（會讓像素風格破功）
- 色板建議維持在 32 色以內（與角色風格一致）

### 透明度

- 所有素材**必須有 alpha 通道**（RGBA PNG）
- 邊緣不要殘留淡灰色（常見於 Photoshop 匯出時的殘影）

### 視角

- **Low Top-down（微俯視）**，與角色視角一致
- Tileset 的地面為**平面俯視**，牆體與 Prop 為**微側視**

## ✅ 提交前自檢清單

送出前請確認：

- [ ] 檔案放在正確的資料夾（tilesets / props 分開）
- [ ] 檔名符合規範（英文、小寫、底線分隔）
- [ ] Tileset 尺寸為 32 的倍數，每格對齊網格
- [ ] Prop 背景完全透明，腳底對齊圖片底部中央
- [ ] 用 Godot 預覽一次（或請程式組協助預覽）沒有出現模糊、錯位

## 📤 如何提交

### 方式 A：直接 commit（推薦給會用 Git 的成員）

```bash
# 將素材放入對應資料夾後
git add game/assets/textures/environment/
git commit -m "新增政大校園 tileset"
git push
```

### 方式 B：打包傳給程式組

若不熟悉 Git，請：
1. 將素材按照本文件的資料夾結構整理好
2. 壓縮成 zip 傳給程式組
3. 附上簡短說明（新增了什麼、替換了什麼）

---

## 🔧 程式組端處理流程（僅供參考）

美術提交素材後，程式組會：

1. 將 tileset PNG 匯入 Godot，建立 `TileSet` 資源（`src/maps/tilesets/<zone>_tileset.tres`）
2. 為 Prop 建立對應的場景（`src/maps/props/<category>/<prop>.tscn`）
3. 在 Zone 場景中擺放 tile 與 prop

美術組**不需要**操作 Godot，只需提供素材檔案即可。
