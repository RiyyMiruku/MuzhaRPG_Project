理念動機

問題分析

在台北唸書／生活、對所在地不熟的年輕族群，對「自己每天經過的街區」缺乏情感連結與在地知識。在地故事散落在長輩記憶、市場攤販的日常裡，沒有被轉成年輕人會主動接觸的媒介。


5W1H · POV

  • When：等放榜、課後、通勤的空白時段
  • Where：木柵市場、政大、指南宮、河濱、貓空

使用者歷程地圖 · PERSONA

王同學，22 歲

政大研究所考生，剛考完研究所、等放榜。住木柵 4 年，每天經過木柵市場去學校，但只去過 3 家店。喜歡敘事型獨立遊戲，會主動關注獨立工作室。

焦慮空白期 在地疏離感 敘事 RPG 受眾
HMW · How might we...

我們能不能用一款 RPG，讓年輕人在等放榜的空白時間裡，重新認識自己生活的這個區？

解決方案

以三條 AI 軸線解構傳統 RPG 開發的成本牆，讓三人團隊也能撐起在地題材的長篇敘事。

01 · 在地化敘事

真實木柵為舞台

以政大、市場、指南宮、河濱、貓空 5 個真實場景重建，玩家認得出每一條巷弄。

02 · 本地端 SLM 對話

每個 NPC 都會聊天

Qwen3.5-0.8B 量化後 531MB，玩家離線可玩，無 API 費用、保護隱私。

03 · AI 美術 PIPELINE

三人撐起 RPG 量級

Pixellab + 自製 orchestrator + Godot 自動 import，1 個 NPC 從 prompt 到入庫 < 10 分鐘。

AI 工作流程

A · 對話系統

三層架構：authored beats（劇本鎖定）→ constrained free chat（角色約束）→ fallback。Prompt 最佳化使用 chatml + assistant prefill <think></think>。

B · 美術 PIPELINE

Pixellab API → 4 種 orchestrator（autotile / prop / NPC static / NPC moving）→ frame-bake spritesheet → Godot .import + .tres 自動產出。

C · WEB DASHBOARD

React + Vite + FastAPI。建立、預覽、重生成、批次任務管理，三人團隊共用協作。

SWOT 分析

S

優勢（內部）

  • 在地題材獨特性：木柵寫實 RPG 為中文 RPG 高度差異化題材
  • 在地共鳴形成天然行銷：政大校友、文山區居民構成核心客群
  • 在地商家合作護城河：後進者難複製，可延伸實體聯名
  • 三人團隊決策快速：無大團隊溝通成本
  • 買斷制商業模式單純：無內購、無營運壓力
W

劣勢（內部）

  • 資金來源有限：自籌資金限制開發週期與美術品質
  • 三人團隊產能受限：RPG 內容量極大，週期可能拉長
  • 題材天花板偏低：非台北玩家、海外玩家共鳴有限
  • 缺乏成功 IP 經驗：玩家對品牌信任需從零建立
  • 商家合作協調成本高：授權、形象使用需大量時間
O

機會（外部）

  • 台灣獨立遊戲市場成熟：返校、九日證明商業可行
  • 在地文化題材正當紅：華語市場偏好「文化記憶」題材
  • 補助與發行支持可能：文化部輔導金、Gamera Games
  • 在地商家有 PR 動機合作：免費行銷接觸年輕客群
  • 實況／影音傳播紅利：敘事 RPG 是實況主偏好類型
T

威脅（外部）

  • Steam 平台競爭激烈：每日新作多，Wishlist 表現關鍵
  • 台灣獨立 RPG 賽道擁擠：需明確差異化才能搶占注意力
  • 玩家對寫實還原期待高：店家還原度、美術質感標準嚴苛
  • 在地商家異動風險：店家可能結束營業、易主
  • 授權與肖像權爭議：未取得書面授權可能法律糾紛
WT 策略（本期採用）：章節 MVP — 首章節先上架／釋出 Demo，前期收入支撐後續章節，每章完成即備份商家素材，分散資金、產能、商家異動的疊加風險。
